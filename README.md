# 拼豆 DIY 小程序（UniApp）

本项目是拼豆 DIY 业务的用户端，基于 UniApp + Vue3 构建，支持 H5 与微信小程序。核心目标是让用户可以完成“选模板 -> 创作拼豆图 -> 下单购买 -> 订单跟踪”的完整闭环。

## 1. 项目定位与职责

- 面向 C 端用户，承载创作、模板、商城、订单、个人中心等核心流程。
- 与 `perler-beads-backend` 通过 HTTP API 通信，负责数据展示与交互体验。
- 与 `perler-beads-admin` 在业务数据上共用同一后端与数据库。

## 2. 技术栈

- 框架：`@dcloudio/uni-app`（Vue3）
- 构建：`vite`
- 样式：`scss`
- 运行目标：H5、微信小程序（可扩展多端）

## 3. 目录结构（核心）

```text
perler-beads-uniapp/
├─ src/
│  ├─ pages/                 # 业务页面（首页/创作/模板/商城/订单/我的）
│  ├─ api/                   # 接口请求封装
│  ├─ stores/                # 前端状态管理（拼豆画布、云端交互等）
│  ├─ static/                # 静态资源
│  ├─ App.vue
│  ├─ main.js
│  ├─ pages.json
│  └─ manifest.json
├─ package.json
└─ README.md
```

## 4. 主要功能

- 创作中心：画布编辑、颜色选择、模板载入、作品保存。
- 模板库：模板预览、分类/难度筛选、使用次数/点赞能力联动。
- 商城与购物车：商品浏览、加入购物车、结算入口。
- 订单流程：确认订单、优惠券使用、订单列表与状态展示。
- 个人中心：用户信息、我的作品、订单入口、常用功能导航。

## 5. 开发与运行

### 5.1 安装依赖

```bash
npm install
```

### 5.2 本地开发

- H5（推荐日常联调）：

```bash
npm run dev:h5
```

- 微信小程序：

```bash
npm run dev:mp-weixin
```

然后在微信开发者工具中打开 `dist/dev/mp-weixin`。

### 5.3 构建发布

- H5：

```bash
npm run build:h5
```

- 微信小程序：

```bash
npm run build:mp-weixin
```

## 6. 配置说明

- 小程序 AppID：修改 `src/manifest.json` 中 `mp-weixin.appid`。
- 页面路由与导航：修改 `src/pages.json`。
- 后端地址：在 `src/api` 下统一配置请求基地址（请与后端端口保持一致）。

## 7. 与后端联调要求

- 后端默认地址：`http://localhost:3000`
- 关键检查：
  - 登录后 token 是否正常携带。
  - 模板接口 `bead_data` 是否正确返回并可渲染。
  - 订单金额、优惠券、状态流转是否与后端一致。

## 8. 常见问题排查

- 页面空白/数据异常：优先检查接口返回结构是否变化。
- 模板预览不显示：检查 `bead_data` 格式与字段映射。
- 金额显示异常：检查数值字段是否为 `null/字符串`，前端是否做了兜底转换。
- 小程序运行异常：确认微信开发者工具基础库版本与 UniApp 编译版本匹配。

## 9. 建议工作流

1. 先启动后端，再启动小程序端。
2. 优先使用 H5 做快速功能验证，再回归微信小程序端。
3. 任何接口字段新增/变更，同步更新本项目 `api` 与对应页面渲染逻辑。
